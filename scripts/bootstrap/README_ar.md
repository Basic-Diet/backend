# دليل عمليات التهيئة وإعادة ضبط قواعد البيانات (Runbook)

هذا الدليل مخصص لفريق التشغيل لتوضيح دورة حياة إدارة البيانات الخاصة بنظام BasicDiet145، وتشمل مراحل: النسخ الاحتياطي (Backup)، المحاكاة الجافة (Dry-run)، إعادة الضبط (Reset)، البذر (Seeding)، والتحقق من جاهزية النظام (Verification).

---

## 1. النسخ الاحتياطي (Backup)

قبل اتخاذ أي إجراء تعديل أو إعادة ضبط لقاعدة البيانات، يجب أخذ نسخة احتياطية كاملة لضمان إمكانية الاسترجاع في حالات الطوارئ.

```sh
# أخذ نسخة احتياطية باستخدام mongodump
mongodump --uri="mongodb://localhost:27017/basicdiet145" --out=/backup/dumps/`date +"%Y-%m-%d"`
```

---

## 2. المحاكاة الجافة (Dry-run)

قبل تشغيل أي عملية حذف أو تهيئة فعلية، يوصى دائماً بتشغيل وضع المحاكاة الجافة (`--dry-run`). في هذا الوضع، لا يتم إجراء أي تعديل على قاعدة البيانات، بل يقوم النظام فقط بطباعة الخطوات التي سيتم تنفيذها.

```sh
# محاكاة عملية البذر بدون كتابة أي بيانات
npm run bootstrap:data -- --dry-run

# محاكاة عملية إعادة ضبط وحذف قواعد البيانات
node scripts/bootstrap/reset-databases.js --dry-run
```

---

## 3. إعادة ضبط قاعدة البيانات (Safe Database Reset)

عملية إعادة الضبط تقوم بحذف قاعدة البيانات لتبدأ ببيئة نظيفة. تمت حماية هذه العملية بشكل صارم لمنع الحذف غير المقصود:

- **الحماية الأولى**: لن تعمل العملية مطلقاً في بيئة الإنتاج (`NODE_ENV=production`).
- **الحماية الثانية**: يتطلب تشغيلها تمرير متغير البيئة `RESET_DB_CONFIRM=true` كتأكيد صريح على العملية.
- **الحماية الثالثة**: يتم التحقق من اسم قاعدة البيانات لضمان عدم المساس بقواعد البيانات النظامية (`admin`, `local`, `config`).

```sh
# تنفيذ إعادة الضبط الفعلية (يتطلب التأكيد الصريح)
RESET_DB_CONFIRM=true node scripts/bootstrap/reset-databases.js
```

---

## 4. البذر وتهيئة البيانات (Seeding)

بعد تنظيف قاعدة البيانات، يتم تشغيل سكربت التهيئة للبذر بالبيانات الأساسية للنظام (قائمة الطعام، خطط الاشتراك، الإضافات، إعدادات الضريبة، ملف المطعم، ومفاتيح الترقية المميزة).
كافة عمليات البذر مصممة لتكون غير مكررة (Idempotent)، مما يعني أنه يمكن إعادة تشغيلها بأمان لتحديث البيانات دون تكرارها.

```sh
# تشغيل بذر البيانات الكامل لإنشاء العناصر المفقودة (وتفعيل بذر الحسابات الافتراضية)
ALLOW_ACCOUNT_BOOTSTRAP=true npm run bootstrap:data

# تشغيل البذر مع فرض مزامنة كافة الحسابات وعناصر خطط الاشتراك
ALLOW_ACCOUNT_BOOTSTRAP=true BOOTSTRAP_SYNC=true ACCOUNT_BOOTSTRAP_SYNC=true npm run bootstrap:data -- --sync
```

---

## 5. التحقق من جاهزية النظام (Verification)

بعد اكتمال تهيئة البيانات، يجب تشغيل سكربت التحقق الآلي للتأكد من أن كافة العقود والمتطلبات البرمجية جاهزة للعمل بشكل سليم.

```sh
# تشغيل التحقق من جاهزية النظام
node scripts/bootstrap/verify-bootstrap-readiness.js
```

التحقق يقوم باختبار 9 نقاط أساسية:
1. وجود مفاتيح الترقية المميزة وتسعيرها (`beef_steak`, `shrimp`, `salmon`, `premium_large_salad`).
2. التأكد من أن مسار قائمة الطعام يعرض الأسعار المميزة بشكل صحيح (> 0).
3. وجود خيارات الإضافات وتسعيرها.
4. وجود خطط الاشتراك الأساسية النشطة (الأيام 7، 14، 21، 28).
5. وجود إعدادات ضريبة القيمة المضافة (`vat_percentage`).
6. وجود عنوان وملف المطعم.
7. وجود نطاقات التوصيل (`Zone`).
8. وجود ساعات عمل المطعم.
9. وجود الحسابات الافتراضية الصالحة للأدوار الإدارية المختلفة.

إذا ظهر التقرير بنجاح (PASS)، فإن النظام يكون جاهزاً للعمل التام.
